Python regex for markdown

tags:

(?<=\s)#[a-zA-Z0-9_-]+

level 1 headers:

# .+
(?<![^\n])# .+

headers of any level:

(^#{1,6} .+)

triple backtick code blocks, with the entire code block as group 1:

(?<=\n)(`{3}(.|\n)*?(?<=\n)`{3})

single backtick code blocks, with the entire code block as group 1:

(`[^`]+?`)

When working with both types of code blocks, you will probably want to handle the triple backtick type first.

markdown-style links:

\[(.+)]\((.+)\)

markdown-style links to markdown zettelkasten files:

\[§\]\((\d{14})\.md\)

zettelkasten-style links:

\[\[(\d{14})\]\]

pdf or png file names and extensions in markdown-style links:

(?<=(\\|/))[^\\/]+\S\.(pdf|png)(?=\))

this also gets some website URLs

pdf or png file names and extensions in markdown-style links, where the file path must be absolute:

(?<=]\(C:)[^\n]+([^\\/\n]+\.(pdf|png))(?=\))

pdf or png file paths in markdown-style links, with the entire match as group 1:

(?<=]\()(.+\S\.(pdf|png))(?=\))

this also gets some website URLs

absolute pdf or png file paths in markdown-style links, with the file names as group 1:

(?<=]\()C:[^\n]*?([^\\/\n]+\.(pdf|png))(?=\))

when using re.sub or re.subn on strings that may contain undefined escapes

"Unknown escapes of ASCII letters are reserved for future use and treated as errors" (source).
To prevent errors and use the literal characters, escape the backslashes with str.replace('\\', r'\\') before calling re.sub or re.subn.
For example, result = re.sub(r'replace this pattern', with_this_str.replace('\\', r'\\'), in_this_str)
