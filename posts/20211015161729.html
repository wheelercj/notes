intro to SQL

  1. Learn SQL with SQLBolt. These exercises can take a few weeks to get through, depending on your schedule.
  2. Choose a database technology that uses SQL. For Python, Full Stack Python compares some of them and has guides for them.
    • Applications that will be downloaded and need a local database appear to usually use SQLite so that the user doesn't have to set up a database themselves.
    • If you choose PostgreSQL, one of the easiest ways to get started with a real PostgreSQL database is to follow this guide to set up a free one on Heroku, and then use Beekeeper Studio (which is free and open source) to interact with the database. Beekeeper Studio even has a special option for setting up with a database hosted on Heroku.
    • If you are making an asynchronous application, you will probably want to choose an asynchronous library for interacting with your database. E.g. asyncpg is a good choice for PostgreSQL in asynchronous Python.
  • There are some SQL services available in the GitHub student developer pack.
