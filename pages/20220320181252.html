how to create custom terminal commands

Creating custom terminal commands can change having to type something long like this:

py -3.10 "C:\Users\chris\Documents\programming\refactor todo list\create_task.py"

to something like this:

todo

And it will work no matter which directory the terminal is in. How to create custom commands depends on your operating system.

Mac and Linux

This guide seems well-written (no need to rewrite the wheel).

Windows

I recommend checking out Windows Terminal if you're choosing a terminal app. Here's how to create custom commands:

  1. Create a folder that will hold the files that describe what your command(s) will do. You can name this folder anything and put it anywhere you want; I happened to name mine C:/Users/chris/Documents/programming/terminal_bindings.
  2. Add the path to the new folder to the PATH user environment variable. See [ยง] how to edit PATH in Windows 10.

The steps above never need to be repeated when you create more terminal commands in the future.

  1. In the new folder, create a script file (e.g. a file that ends with .py if you want to use Python, or .bat or .cmd if you want to use Batch scripts, or .ps1 if you would prefer to use PowerShell commands). The file name must start with what you want the terminal command to be. For example, if you want to create a project command that uses Batch to open a bunch of applications and/or websites at once for a project you're working on, create project.bat.
  2. Put terminal command(s) into this new file. Continuing with the above example, the contents of the file could be something like this Batch file I use when I start remote tutoring sessions:
@echo off
start "" "https://web.penjiapp.com/communities/xcvnssap3/teach/drop-in-waitlist/vosiueu8c/hours/ovxueaqodm"
start "" "C:/Users/chris/AppData/Local/Microsoft/Teams/current/teams.exe"
start "first zoom" C:\Users\chris\AppData\Roaming\Zoom\bin\zoom.exe
start "second zoom" C:\Users\chris\AppData\Roaming\Zoom\bin\Zoom1.exe
"C:/Users/chris/AppData/Roaming/Microsoft/Windows/Start Menu/Programs/zq/zq.lnk"

Now you can run your script by simply opening your terminal and typing the command you created without worrying what the current working directory is!

some other scripts that I use

todo.bat

@echo off
py -3.10 "C:\Users\chris\Documents\programming\refactor todo list\create_task.py" "%*"

The "%*" at the end allows the Python script to receive any number of words as arguments when I use the todo command. For example, I could enter todo buy oranges and apples to add "buy oranges and apples" to my to do list.

venv.ps1

venv/Scripts/Activate.ps1

This makes it easy to activate a Python virtual environment for PowerShell named venv in the current working directory.

binds.bat

@echo off
explorer C:\Users\chris\Documents\programming\terminal_bindings

This opens the folder I keep these files in.

radix.bat

@echo off
"C:\Users\chris\Documents\programming\Radix-Converter\Debug\Radix-Converter.exe"

This is the command to run the exe for a C++ program I wrote.

rq.bat

@echo off
if [%1]==[] goto :noarg
py -3.9 C:/Users/chris/Documents/programming/run-quick/main.py %1
goto :eof
:noarg
py -3.9 C:/Users/chris/Documents/programming/run-quick/main.py

Here's a script that takes one or zero arguments and calls another script with the correct number of arguments.
