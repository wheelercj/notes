how to create custom terminal commands

Creating custom terminal commands can change having to type something long like this:

py -3.10 "C:\Users\chris\Documents\programming\refactor todo list\create_task.py"

to something like this:

todo

And it will work no matter which directory the terminal is in. How to create custom commands depends on your operating system.

Mac and Linux

This guide seems well-written (no need to rewrite the wheel).

Windows

I recommend checking out Windows Terminal if you're choosing a terminal app. Here's how to create custom commands:

  1. Create a folder that will hold the files that describe what your command(s) will do. You can name this folder anything and put it anywhere you want; I happened to name mine C:/Users/chris/Documents/programming/terminal_bindings.
  2. Add the path to the new folder to the PATH user environment variable. See [ยง] how to edit PATH in Windows 10.

The steps above never need to be repeated when you create more terminal commands in the future.

  1. In the new folder , create a .bat file named after what you want the terminal command to be. For example, if you want to create a todo command that adds tasks to a to do list by running a Python script (like this one), create todo.bat.
  2. Put into this new file the terminal command(s) that will run the script/app. Continuing with the above example, the contents of the file could be py -3.10 "C:\Users\chris\Documents\programming\refactor todo list\create_task.py" "%*". If you want to run an .exe file, you only need to enter the path to that file, surrounded by quotes if there are spaces in the file path.

Now you can run your script by simply opening your terminal and typing the command you created without worrying what the current working directory is.

sample .bat files that I use

The contents of .bat files run as if they had been typed into Command Prompt.

todo.bat

py -3.10 "C:\Users\chris\Documents\programming\refactor todo list\create_task.py" "%*"

The "%*" at the end allows the Python script to receive any number of words as arguments when I use the todo command.

tutor.bat

py -3.10 C:\Users\chris\Documents\programming\opener\main.py
start /d "C:/Users/chris/AppData/Local/Microsoft/Teams/current" Teams.exe
start /d "C:/Users/chris/AppData/Roaming/Zoom/bin" Zoom.exe
timeout 1
C:/Users/chris/Documents/programming/terminal_bindings/zq.bat

This command starts four apps I use while I'm tutoring. There's a timeout 1 to pause the script for a second so that an error message from Microsoft Teams doesn't mess up the output from what runs after it. I decided to use start /d as the start of the commands to run executable files because without that, there were some cases where one of them would freeze and prevent the rest of the file from running.

radix.bat

"C:\Users\chris\Documents\programming\Radix-Converter\Debug\Radix-Converter.exe"

Here's the exe for a C++ program I wrote. No need for the start /d here.

zq.bat

py -3.10 C:\Users\chris\Documents\programming\zq\src\zq\main.py

binds.bat

explorer C:\Users\chris\Documents\programming\terminal_bindings

This opens the folder I keep these files in.
