intro to SQL

SQL (Structured Query Language) is a commonly used and reliable way for programs to save and load data. For large amounts of data, it's usually much better than just using a text file.

There's a lot that can be learned about SQL. Here are some resources to get you started. (I have no affiliation with any of these.)

  1. Learn SQL with SQLBolt. These exercises can take a few weeks to get through, depending on your schedule.
  2. Choose a database technology that uses SQL.
    • For Python, Full Stack Python compares some of them and has guides for them.
    • Supabase and Turso (and some others) have decent free tiers, make setting up databases relatively easy, and are open source.
    • Applications that will be downloaded and need a local database appear to usually use SQLite so that the user doesn't have to set up a database themselves. Here's a 2022/4/25 HN discussion on the pros and cons SQLite vs other database technologies.
    • Beekeeper Studio is a free and open source SQL editor and database manager. I use it and it seems to work well.
    • If you are making an asynchronous application, you will probably want to choose an asynchronous library for interacting with your database. E.g. asyncpg is a good choice for PostgreSQL in asynchronous Python.
  • There are some SQL services available in the GitHub student developer pack.
